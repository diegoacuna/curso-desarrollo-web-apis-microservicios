¿En qué estamos?

  • Desarrollo web
    • Contraste entre los sistemas cliente-servidor clásicos y nuestro enfoque moderno
    • ¿Qué hace el servidor?
    • ¿Qué hace el cliente?
    • ¿Cuáles son las diferencias en las funciones del cliente y servidor entre el enfoque tradicional con nuestro enfoque moderno?

El servidor

  • Principalmente procesa datos/información
  • Patrón MVC: el servidor (código de la aplicación) es la C en MVC
  • Observación: la frase anterior omite el real significado de "servidor".

El cliente

  • Reconozcamos quien es el cliente
  • ¿Cuál es su función?
  • En los sistemas tradicionales: ¿Qué hace y qué no hace?

Contexto

  • Estamos construyendo aplicaciones web en forma de una API REST
    • Ya no devolvemos "vistas" al cliente (código html que representa una interfaz gráfica con los datos que procesa el servidor)
    • Solo devolvemos los datos procesados por el servidor en forma de algún formato en particular (JSON, XML, etc.)
    • Dejamos el trabajo de desplegar la interfaz gráfica al cliente (navegador web). Las tecnologías del lado del cliente cobran alta importancia (javascript!)

¿Qué estamos haciendo?

  • Vamos a hablar de Backend y Frontend. En nuestro enfoque moderno ahora están realmente separados.
  • BACKEND: nuestra lógica de negocio desde el lado del servidor. En términos prácticos será nuestro código fuente que correra en el servidor: rails (ruby), expressJS (javascript), Play (scala), Django (python), algún framework en php, etc.
  • FRONTEND: nuestra interfaz gráfica con la que el usuario final interactúa, es decir, lo que se ve en el navegador web. La construimos con html, css y javascript.

Proyecto del curso

  • Máximo grupos de 3
  • Desarrollar una aplicación web de mediana complejidad (interacción con una base de datos, uso de alguna API externa como facebook, google, etc)
  • Desarrollo tanto del backend como del frontend.
  • Pueden utilizar las tecnologías que ustedes deseen.
  • El backend debe ser una API REST.
